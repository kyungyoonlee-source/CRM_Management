현재 데이터 구조 및 한계
⚠️ 현재 방식 (localStorage)
• 데이터가 각자 브라우저에만 저장
• 팀원 간 실시간 공유 불가
• PC 바꾸면 데이터 초기화
• 동시 접속 시 충돌 위험
✅ 목표 방식 (공유 DB)
• 모든 팀원이 같은 데이터 실시간 공유
• 어느 PC/폰에서든 동일한 데이터
• 데이터 안전 보관 (클라우드)
• 팀별 접근 권한 분리 가능
🗺️ 데이터 공유 전환 로드맵
STEP 1
즉시
JSON 백업 공유 방식
현재 지원 중. 팀장 PC에서 데이터 입력 → JSON 백업 → 슬랙/이메일로 공유 → 팀원 복원. 완벽하지 않지만 당장 사용 가능한 방법.
STEP 2
1~2주
Google Sheets 연동
현재 Google Sheets로 관리 중인 기관 정보를 이 시스템에서 불러오고 저장. 팀원 모두가 같은 Sheets에 접근 가능. 개발자 없이도 구현 가능.
STEP 3
1~2개월
Supabase (무료 클라우드 DB) 연동
팀원 모두 실시간 공유, 팀별 로그인 권한 분리, 데이터 영구 보관. 무료 플랜으로 시작 가능. 개발자 1명이 반나절이면 셋업 가능.
STEP 4
장기
자체 앱 VOC 자동 연동
자체 앱에서 VOC 발생 시 이 시스템에 자동 등록. OPS팀이 별도 입력 없이 실시간 확인 가능. API Webhook 방식으로 구현.

📁 현장 파일 공유 규칙
원칙: 현장에서 수집된 모든 자료(사진, 메모, 체크리스트)는 방문 당일 CRM에 등록한다.

방법: 방문 기록 탭 → '컨설팅' 또는 '교육' 유형으로 등록 → 내용에 핵심 포인트 요약

금지사항: 개인 메신저에만 공유하고 CRM 미등록하는 행위. 중복 등록 방지를 위해 등록 전 기존 기록 확인 필수.

자체 앱 VOC 연동
연동 예정
어떻게 작동하나요?
자체 앱에서 사용자(기관)가 VOC를 남기면 → 자동으로 이 CRM에 VOC가 등록됩니다.
OPS팀이 앱을 따로 확인하지 않아도 CRM 하나에서 모든 VOC를 볼 수 있습니다.
구현을 위해 개발팀에 전달할 내용
방식: Webhook (HTTP POST 요청)
조건: 앱에서 VOC 등록 시 자동 트리거
전달 데이터: 기관ID, 기관명, VOC내용, 등록일시, 유형
수신 주소: CRM API 엔드포인트 (Supabase 또는 자체 서버)
인증: API Key 방식
💡 지금 당장 할 수 있는 것: 앱 VOC를 CSV로 정기 내보내기 → 이 시스템 CSV 업로드로 수동 동기화 (주 1회 권장)

Google Sheets 연동
연동 예정
어떻게 작동하나요?
현재 Google Sheets에 있는 전국 기관 정보를 이 CRM에서 바로 불러오거나 저장합니다.
Sheets 수정 → CRM 반영, CRM 수정 → Sheets 반영이 자동으로 이루어집니다.
지금 당장 할 수 있는 방법 (3단계)
1
Google Sheets에서 기관 목록을 CSV로 내보내기 (파일 → 다운로드 → CSV)
2
CRM '기관 관리' 탭 → 'CSV 업로드' 버튼으로 가져오기
형식: 기관명, 담당연구원, 지사, 지역, 원장명, 연락처, 연차, 등급, 특징
3
CRM에서 업데이트한 내용은 'CSV 내보내기'로 다운로드 → Sheets에 붙여넣기
🚀 자동 실시간 연동을 원하신다면: Google Sheets API + Apps Script 연동이 필요합니다.
Google Cloud Console에서 Sheets API 활성화 → API Key 발급 → 이 시스템에 연결.
제공 필요 정보: Google Sheets URL(공유 링크), Google 계정
📋 연동 우선순위 권장 순서
1순위 (즉시) — CSV 수동 동기화 (Google Sheets ↔ CRM)
2순위 (1~2주) — Google Sheets API 자동 연동
3순위 (1~2개월) — Supabase 공유 DB 전환 (전팀 실시간 공유)
4순위 (장기) — 자체 앱 Webhook VOC 자동 등록
